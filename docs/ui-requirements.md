# 智行·AI（MeetMind）UI 需求文档（代码对齐版）

> 更新日期：2026-02-27  
> 基线范围：`packages/web` + `packages/api` 当前实现  
> 文档目标：以“已上线代码能力”为准，统一 UI 设计、交互状态与后续补齐方向。

## 1. 产品范围与端形态
- 当前为 Web 应用（Next.js App Router），覆盖桌面/平板/手机响应式体验。
- 具备 PWA 基础：`manifest`、生产环境 Service Worker、离线录音上传队列。
- 覆盖主闭环：会议录音 → AI 纪要/思维导图 → 任务/OKR/日历联动 → 邮件/PPT 输出 → 对话助手操作。

## 2. 信息架构（当前路由）
### 2.1 公共区
- `/`：落地页（能力说明、流程演示、CTA）。

### 2.2 认证区
- `/login`：登录。
- `/register`：注册。

### 2.3 工作台区
- `/meetings`：会议列表。
- `/meetings/[id]`：会议详情（录音、纪要、思维导图、快速操作）。
- `/tasks`：任务看板/列表。
- `/projects`：项目/OKR。
- `/calendar`：日程日历。
- `/translate`：同声传译。
- `/assistant`：对话助手。
- `/settings`：设置（当前为说明型页面）。

## 3. 全局 UI 需求
### 3.1 导航与布局
- 顶部 AppBar 固定，包含：菜单按钮、站内提醒、主题切换、用户菜单。
- 抽屉导航固定包含：会议列表、任务看板、项目/OKR、日程日历、同声传译、对话助手、设置。
- 桌面端使用左侧抽屉；移动端使用底部半屏抽屉。
- 主内容区统一卡片化，移动端优先圆角胶囊按钮。

### 3.2 主题与视觉
- 支持明暗主题切换，状态持久化到 `localStorage`（`theme_mode`）。
- 暗色模式下 Button/Paper/Drawer/Menu/Chip 具备单独配色策略。
- 全局反馈统一使用 Snackbar + Alert（右上角堆叠）。

### 3.3 站内提醒
- 提醒来源为任务到期窗口（未完成且 7 天内到期，最多 6 条）。
- 提醒支持“已读持久化”，按用户维度存储。
- 点击提醒跳转目标：会议详情或任务高亮定位。

### 3.4 全局交互规范
- 删除类操作需要二次确认（统一 `ConfirmDialog`）。
- 长耗时流程必须有状态反馈（会议分析、PPT 生成）。
- 失败场景提供明确错误提示，避免静默失败。

### 3.5 PWA 与离线
- 生产环境注册 Service Worker，缓存核心静态资源。
- 录音导入支持离线入队（IndexedDB），联网后自动补传。

## 4. 页面级需求（与代码一致）
### 4.1 落地页 `/`
- 展示核心价值、流程链路、能力卡片与“真实流程演示”动画区。
- 登录态用户显示“进入工作台”；未登录显示“免费开始/登录”。
- 页面底部提供二次 CTA 与版权区。

### 4.2 登录 `/login`
- 字段：用户名（或邮箱）、密码。
- 校验：必填校验；失败提示“用户名/密码错误”。
- 成功后写入登录态并跳转 `/meetings`。
- 支持主题切换按钮。

### 4.3 注册 `/register`
- 字段：用户名、邮箱、密码、确认密码。
- 校验：必填、密码长度（>=6）、两次密码一致。
- 成功后自动登录并跳转 `/meetings`。
- 提供“已有账号，去登录”入口。

### 4.4 会议列表 `/meetings`
- 列表卡片展示：标题、时间、参会人数、录音状态、纪要状态、标签。
- 支持分页（每页 12 条）。
- 支持删除会议（二次确认）。
- 支持“新建会议”弹窗：当前字段为标题+标签。
- 支持“导入录音创建会议”弹窗：上传音频并新建会议（支持 MP3/M4A/WAV/AAC）。

### 4.5 会议详情 `/meetings/[id]`
#### 4.5.1 顶部信息区
- 显示会议标题、时间、标签、录音/纪要状态。
- 支持编辑会议（标题、时间）。
- 支持返回会议列表。

#### 4.5.2 Tab：录音
- 展示录音播放器列表（播放/暂停/进度/静音/seek）。
- 无录音时显示空态，引导导入录音。
- 支持导入录音弹窗，离线时自动进入补传队列。

#### 4.5.3 Tab：纪要
- 纪要模块显示：摘要、情绪与风险、关键决策、风险问题、转写内容。
- 移动端纪要区支持子 Tab：会议纪要/待办事项/会议语音转写。
- 任务模块（`MeetingTasks`）支持：
- 按会议筛选任务列表。
- 同步纪要行动项到任务（去重）。
- 任务新增/编辑/删除。
- 任务状态切换（勾选完成、状态菜单）。
- 支持“生成 OKR”（基于会议纪要生成项目/目标/KR 后跳转项目页）。
- 支持会议记忆：
- 录入记忆条目（标题可空，内容必填）。
- 列表查看历史记忆。

#### 4.5.4 Tab：思维导图
- 当有纪要时支持生成思维导图。
- 展示 ReactFlow 导图（自动布局/缩放控制）。
- 右侧聊天面板支持自然语言编辑导图。
- 无纪要时显示空态提示。

#### 4.5.5 Tab：快速操作
- 操作项包含：
- 导入录音。
- 生成纪要（含进度、阶段、ETA）。
- 导入日历（下载会议 ICS）。
- 系统日历订阅（获取 WebCal 地址）。
- 生成邮件草稿。
- 生成 PPT。
- 导出纪要。
- 禁用规则：
- 无录音：禁用“生成纪要”。
- 无纪要：禁用导入日历、生成邮件、生成 PPT、导出纪要。

#### 4.5.6 纪要导出弹窗
- 支持导出格式：`Markdown(.md)`、`Text(.txt)`、`Word(.doc)`。

#### 4.5.7 邮件草稿弹窗
- 字段：收件人（逗号分隔）、主题、正文。
- 支持发送邮件与复制邮件内容。
- SMTP 未配置时弹出说明弹窗并引导至设置页。

#### 4.5.8 PPT 弹窗（重点）
- 三个子 Tab：预览、编辑、背景。
- 预览：
- 状态标签 + 进度条 + 阶段节点 + 生成日志。
- 支持 HTML/PDF 预览切换。
- 支持缩略图导航（图像或提纲）。
- 支持下载 PDF/PPTX。
- 编辑：
- 加载并编辑 Markdown。
- 提供格式工具（粗体/斜体/下划线/列表/标题/字号/颜色/对齐）。
- 支持语音输入补充内容。
- 支持按页重排、单页标题与内容编辑。
- 背景：
- 全局背景配置。
- 素材源切换：素材库/已上传/AI 推荐。
- 支持上传背景图（jpg/jpeg/png/webp）。
- 支持单页背景覆盖（跟随全局或单独选择）。

### 4.6 会议路由悬浮录音按钮
- 会议域布局下存在全局悬浮录音按钮（RecordFab）。
- 行为：开始录音 → 停止录音后自动创建会议并上传录音 → 跳转新会议详情。

### 4.7 任务看板 `/tasks`
- 支持看板视图与列表视图。
- 桌面看板支持 4 列（待处理/进行中/已完成/阻塞）拖拽流转。
- 移动端支持 Tab 切换与单列视图。
- 支持搜索（标题/负责人）。
- 支持统计筛选（全部、待处理、进行中、已完成、逾期）。
- 支持任务高亮定位（通过 `?highlight=`）。
- 任务卡支持：
- 编辑、删除、完成。
- 会议来源跳转。
- 音频锚点回放（跳转会议并 `seek`）。
- 任务弹窗支持字段：标题、描述、负责人、截止日期、优先级、关联 KR。

### 4.8 项目/OKR `/projects`
- 展示项目列表、状态、起止日期、总进度。
- 支持项目 CRUD。
- 支持目标（Objective）CRUD。
- 支持关键结果（Key Result）CRUD。
- 支持目标折叠/展开。
- 显示 Objective/KR 进度条与状态标签。
- 支持“关联任务到 KR”：
- 按关键结果选择任务并批量绑定。
- 支持任务搜索与已关联提示。
- 支持“创建项目时导入任务”：
- 从任务看板选择任务。
- 自动生成目标与 KR，并回写任务关联。

### 4.9 日程日历 `/calendar`
- 基于 FullCalendar 展示会议+任务统一视图。
- 视图：月、周、日、列表（移动端默认列表周视图）。
- 点击会议跳转会议详情，点击任务跳转任务高亮。
- 提供“订阅到系统日历”按钮（WebCal）。

### 4.10 同声传译 `/translate`
- 支持文字输入与语音输入双模式。
- 支持源语言自动识别开关。
- 支持 AI 增强翻译开关。
- 支持多语言互译（中英日韩法德西俄葡意阿印等）。
- 支持交换语种（自动识别开启时禁用）。
- 输出区支持复制译文、语音朗读译文。

### 4.11 对话助手 `/assistant`
- 支持会话列表（创建、切换、删除），会话历史持久化。
- 移动端会话列表为底部抽屉。
- 支持机器人（Agent）选择、创建、编辑、删除。
- 移动端支持机器人选择抽屉与管理入口。
- 支持文本输入与语音输入。
- 助手消息支持 Markdown 渲染。
- 支持建议短语（Chip）点击追问。
- 支持组件化结果卡片类型：
- 会议列表。
- 任务列表。
- 创建会议表单。
- 思维导图。
- PPT 进度与下载。
- 邮件草稿。
- 日程事件。
- 分析进度。
- 下载文件。
- 对 `analysis/slides` 类型消息执行自动轮询刷新状态。

### 4.12 设置 `/settings`
- 当前页面为说明型配置页，包含：
- 三个本地开关文案（自动分析、自动提取任务、高风险确认）。
- 第三方服务连接状态展示（静态）。
- SMTP 配置说明输入框（仅展示说明，不持久化）。

## 5. 与 UI 相关的后端能力要求（已存在）
- 认证：注册/登录/当前用户查询。
- 会议：增删改查 + 分页列表。
- 录音：会议录音列表、音频导入。
- 分析：启动分析、查询进度、获取纪要、生成/编辑导图、导图聊天历史。
- 任务：看板查询、创建、更新、完成、删除，支持会议过滤与 OKR 关联字段。
- 项目/OKR：从会议生成项目、项目/目标/KR 全量 CRUD、KR 完成联动任务完成。
- 日历：统一事件查询、会议 ICS 下载、用户级 WebCal 订阅 feed。
- 邮件：纪要邮件草稿生成与发送。
- PPT：生成、状态、日志、导出、预览、Markdown 编辑、背景素材与推荐、导出可用性检查。
- 助手：会话管理、机器人管理、工具化调用、建议语句。
- 记忆：会议记忆创建/列表、跨会议记忆搜索（后端已提供）。

## 6. 状态与边界处理要求
- 空状态：会议、任务、纪要、导图、PPT、提醒、聊天历史均需空态文案。
- 加载状态：列表加载、分析进度、PPT 进度、助手思考中、日历加载中。
- 错误状态：
- API 失败需 toast 或明确错误文案。
- 邮件发送失败且包含 SMTP 错误需进入专用提示弹窗。
- 语音识别不可用需提示浏览器不支持。
- 条件禁用：
- 所有依赖前置产物的操作按钮必须禁用（如无纪要不可生成邮件/PPT）。

## 7. 响应式与交互一致性要求
- 断点基于 MUI `sm`（约 600px）进行移动端切换。
- 移动端策略：
- 抽屉多使用底部弹出。
- 关键弹窗可全屏。
- 主操作按钮尽量全宽与胶囊化。
- 桌面端策略：
- 支持多列看板与侧栏并行信息呈现。
- PPT 预览/编辑支持缩略图 + 主视图双栏。

## 8. 已知缺口（需纳入后续 UI 迭代）
- 设置页当前为静态说明，尚未接入真实配置读写。
- 前端尚缺统一路由级登录守卫（目前主要依赖 API 鉴权失败反馈）。
- 会议新建弹窗字段较简化（未覆盖参与人/时间/时区完整输入）。
- `memory/search` 能力已在后端提供，前端暂无独立检索入口。
- 助手工具清单可查看，但工具可视化编排与可解释性仍可增强。
