# 第一章 概 述

## 1.1 项目背景

在混合办公与移动办公日益普及的今天，会议依然是企业协作的核心场景。然而，传统的会议模式面临着严重的“信息熵增”问题：海量的语音信息在转化为执行动作的过程中存在巨大的损耗。据统计，管理者平均每天花费30%—50%的时间在会议上，但会议纪要整理滞后、待办任务遗漏、决策背景模糊等问题，导致会议产出与业务执行之间存在明显的效率断层。

目前市面上的主流会议AI工具（如Otter.ai、通义听悟等）大多停留在“被动记录者”的角色。它们虽然能利用AIGC技术实现高精度的语音转写和摘要生成，但往往止步于生成文本。用户仍需手动将摘要复制到日历、将待办录入任务软件、手动制作PPT汇报。这种“工具孤岛”现象使得移动端的办公体验割裂，无法满足用户在通勤、差旅等碎片化场景下即时处理的需求。

随着大模型技术向智能体演进，以及移动终端NPU算力的提升，AI已具备从内容生成向任务执行跨越的能力。本项目旨在抓住这一机遇，利用端云协同技术，打造一款能够主动理解意图、闭环执行任务的下一代移动办公助理。

## **1.2 项目目的**

本项目的核心目的是打造“听-记-行-查”的主动执行型智能体，  
重构会议办公流程，从单一的音频处理升级为全链路的业务闭环。系统不仅要“听懂”会议，更要能够“执行”决策。通过深度语义分析，自动完成从非结构化音频到结构化日历日程、邮件草稿、待办看板的转化，实现会议结束，部署完成。

其次，针对移动端操作不便的痛点，本项目致力于探索多模态交互在办公场景的落地。通过“语音指令构建框架+手势微调细节”的创新模式，让用户在手机上也能高效生成和修改PPT、处理复杂邮件。

最后，为解决任务执行中的责任归属与事件遗忘问题，通过建立音频与任务的毫秒级映射生成音频锚点，以及基于时序知识图谱的项目记忆库，让每一个待办都能追溯到原始语音，让每一次决策都能关联历史背景，提升团队协作的透明度与连贯性。

## 1.3 项目创新点

本项目区别于传统的会议记录工具，在业务闭环、交互体验与技术深度三个维度进行了突破性创新。

业务模式创新点一：构建“听-记-行-查”的全链路办公闭环，从文本生成到系统级执行，突破传统工具仅生成文本摘要的限制，本系统作为主动执行型Agent，能自动提取会议中的时间、地点、责任人，直接调用系统API生成日历日程、待办清单及邮件草稿。用户无需在多个APP间跳转，极大降低了人工操作成本。

业务模式创新点二：针对任务执行中来源不清、语境丢失的痛点，建立结构化任务与非结构化音频的强关联。在待办看板中，点击任意任务即可毫秒级定位并回放该任务对应的原始会议录音片段，让任务确认有据可依。

交互体验创新点一：针对移动端编辑PPT效率低下的痛点，提出创新的多模态交互方案。用户通过语音指令快速构建PPT框架，并通过移动端原生手势（双指缩放排版、长按拖拽节点）进行细节调整，实现高效创作。

交互体验创新点二：端云协同的离线智能助理，适配高铁、电梯等弱网/无网环境，利用端侧模型实现离线录音与基础意图识别。结合自然语言交互（NLP），用户可像与真人秘书对话一样下达指令，系统自动调取上下文执行，确保全场景服务。

技术深度创新点一：基于时序知识图谱的长时记忆构建以项目为核心的知识库。智能体能够关联历史会议内容，如自动检索上次会议遗留的技术难题，减少重复沟通，提升AI辅助决策的逻辑连贯性。

技术深度创新点二：引入情感计算模型，融合音频声学特征（语调、语速、停顿）与文本语义，识别会议中的犹豫点、争论点或消极情绪。在生成会议纪要时，自动对高风险任务进行标记，辅助管理者进行风险预判。
